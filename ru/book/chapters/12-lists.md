## Списки

**Списки (Lists)** в **Markdown** позволяют структурировать информацию и делать документацию более понятной.

В технической документации и учебных материалах по **SQL** списки часто используются для перечисления команд, этапов работы с базой данных или типов операций.

### Неупорядоченные списки

**Неупорядоченные списки (Unordered Lists)** создаются с использованием символов `-`, `*` или `+`.

**Пример (Markdown):**

```markdown
- SELECT — получение данных из таблицы
- INSERT — добавление новых записей
- UPDATE — изменение существующих данных
```

**Результат (HTML):**

```html
<ul>
  <li>SELECT — получение данных из таблицы</li>
  <li>INSERT — добавление новых записей</li>
  <li>UPDATE — изменение существующих данных</li>
</ul>
```

### Упорядоченные списки

**Упорядоченные списки (Ordered Lists)** создаются с использованием чисел и точки.

Они часто применяются для описания последовательности действий при работе с базой данных.

**Пример (Markdown):**

```markdown
1. Подключиться к базе данных
2. Написать SQL-запрос
3. Выполнить запрос и проверить результат
```

**Результат (HTML):**

```html
<ol>
  <li>Подключиться к базе данных</li>
  <li>Написать SQL-запрос</li>
  <li>Выполнить запрос и проверить результат</li>
</ol>
```

### Вложенные элементы списка

Элементы списка могут содержать вложенные списки. Это удобно, когда нужно структурировать информацию о таблицах, полях или типах данных.

**Пример (Markdown):**

```markdown
- **Таблица users**
  - id
  - name
  - email
- **Таблица orders**
  - id
  - user_id
  - total_price
```

**Результат (HTML):**

```html
<ul>
  <li>
    <strong>Таблица users</strong>
    <ul>
      <li>id</li>
      <li>name</li>
      <li>email</li>
    </ul>
  </li>
  <li>
    <strong>Таблица orders</strong>
    <ul>
      <li>id</li>
      <li>user_id</li>
      <li>total_price</li>
    </ul>
  </li>
</ul>
```
