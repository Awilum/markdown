## Оглавление

В **Markdown** нет специального синтаксиса для создания оглавления. Обычно оно создаётся с помощью **ссылок**, которые ведут на **заголовки** документа. Такой способ позволяет быстро переходить к нужным разделам в длинных статьях, инструкциях, учебных материалах и документации.

### Простейшее оглавление на основе заголовков

**Пример (Markdown):**

```markdown
## Оглавление

- [Введение](#введение)
- [Установка](#установка)
- [Использование](#использование)
- [Частые вопросы](#частые-вопросы)

## Введение
Текст раздела...

## Установка
Текст раздела...

## Использование
Текст раздела...

## Частые вопросы
Текст раздела...
```

**Результат (HTML):**

```html
<h2>Оглавление</h2>
<ul>
  <li><a href="#введение">Введение</a></li>
  <li><a href="#установка">Установка</a></li>
  <li><a href="#использование">Использование</a></li>
  <li><a href="#частые-вопросы">Частые вопросы</a></li>
</ul>

<h2 id="введение">Введение</h2>
<h2 id="установка">Установка</h2>
<h2 id="использование">Использование</h2>
<h2 id="частые-вопросы">Частые вопросы</h2>
```

### Как формируются ссылки на заголовки (якоря)

Ссылки в оглавлении ведут к заголовкам документа.

Обычно якорь автоматически формируется из текста заголовка по следующим правилам:

-   все буквы преобразуются в нижний регистр;
-   пробелы заменяются символом `-`;
-   большинство специальных символов удаляется.

**Пример (Markdown):**

```markdown
## Как установить программу

[Как установить программу](#как-установить-программу)
```

**Результат (HTML):**

```html
<h2 id="как-установить-программу">Как установить программу</h2>

<a href="#как-установить-программу">Как установить программу</a>
```

> **Обратите внимание.** 
> 
> Конкретные правила формирования якорей могут отличаться в зависимости от используемого Markdown-редактора или платформы. Например, в GitHub, GitLab, Pandoc, MkDocs и других системах они реализованы по-разному. Для русского заголовка `## Введение` один рендерер может создать якорь `#введение`, другой — использовать URL-кодирование, третий — транслитерацию, а некоторые вообще не поддерживают русские якоря.
> 
> Поэтому, если документ должен одинаково работать на разных платформах, рекомендуется использовать английские идентификаторы или, если это поддерживается используемым Markdown-движком, явно задавать идентификаторы для заголовков.

### Типичные ошибки при создании оглавления

-   ссылка не совпадает с якорем заголовка;
-   используются заглавные буквы вместо строчных;
-   вместо символа `-` используются пробелы;
-   оглавление не обновлено после изменения заголовков.

Неправильно:

```markdown
[Установка](#Установка)
```

Правильно:

```markdown
[Установка](#установка)
```

### Где используется оглавление

-   README-файлы проектов;
-   техническая документация;
-   учебные материалы;
-   инструкции и руководства.

Если документ содержит много разделов, оглавление значительно упрощает навигацию и делает материал более удобным для чтения.
